1. Implement `tests/shadow/src/scenarios/cs_api.rs` with a single entry point:

   ```rust
   pub async fn run_cs_api(server_url: &str, role: &str) -> Result<(), Box<dyn std::error::Error>>
   ```

   This dispatches to `run_alice(server_url)` or `run_bob(server_url)` based on role.

   **Alice flow** (per D-05, D-08):
   - `create_sdk_client(server_url).await?` (waits for server readiness)
   - `register_with_token(&client, "alice", DEFAULT_PASSWORD, REGISTRATION_TOKEN).await?` (TEST-01)
   - `login_user(&client, "alice", DEFAULT_PASSWORD).await?` (TEST-02)
   - Create room with alias: use `matrix_sdk::ruma::api::client::room::create_room::v3::Request`. Set `room_alias_name = Some("test-room".to_owned())`. Call `client.create_room(request).await?`. Log room_id to stderr. (TEST-03)
   - Send message: get the Room from the response, call `room.send(RoomMessageEventContent::text_plain("Hello from Alice")).await?`. Log to stderr. (TEST-04)
   - `eprintln!("alice: cs-api scenario complete")`

   **Bob flow** (per D-05, D-08, D-09):
   - `create_sdk_client(server_url).await?` (waits for server readiness)
   - `register_with_token(&client, "bob", DEFAULT_PASSWORD, REGISTRATION_TOKEN).await?`
   - `login_user(&client, "bob", DEFAULT_PASSWORD).await?`
   - Join room by alias with retry loop (per research Pitfall 3, D-09): attempt `client.join_room_by_id_or_alias(alias, &[])` in a loop with up to 30 retries, 1000ms `tokio::time::sleep` between attempts. The alias is `#test-room:tuwunel-server`. Parse alias using `<&RoomOrAliasId>::try_from("#test-room:tuwunel-server")`. (TEST-05)
   - Sync to receive messages: `client.sync_once(SyncSettings::default()).await?`
   - Verify message received: check the joined room's timeline in the sync response for a message containing "Hello from Alice". Log result to stderr.
   - `eprintln!("bob: cs-api scenario complete")`

   Use `matrix_sdk::ruma::events::room::message::RoomMessageEventContent` for the message.
   Use `matrix_sdk::config::SyncSettings` for sync.
   Use `matrix_sdk::ruma::RoomOrAliasId` for the alias type.

   Per research anti-pattern: use `tokio::time::sleep` (not `std::thread::sleep`) for all waits.
   Per research Pitfall 6: use `sync_once()` not infinite `sync()`.

2. In `tests/shadow/src/bin/matrix_test_client.rs`, replace the CsApi match arm placeholder with:
   ```rust
   Commands::CsApi { server_url, role } => {
       let rt = tokio::runtime::Builder::new_current_thread()
           .enable_all()
           .build()
           .expect("failed to build tokio runtime");
       match rt.block_on(shadow_test_harness::scenarios::cs_api::run_cs_api(&server_url, &role)) {
           Ok(()) => ExitCode::SUCCESS,
           Err(e) => {
               eprintln!("CS API test failed: {e}");
               ExitCode::FAILURE
           },
       }
   },
   ```
