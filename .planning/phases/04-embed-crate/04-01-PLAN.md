  <name>Task 1: Fix OnceLock statics in runtime.rs for multi-call safety</name>
  <files>src/main/runtime.rs</files>
  <read_first>src/main/runtime.rs</read_first>
  <action>
Change three OnceLock `.set().expect()` calls in `runtime::new()` (lines 45-55) to `.get_or_init()`. This makes the function safe to call multiple times -- first caller wins, subsequent calls are no-ops. Per D-01 and EMBD-08.

Specifically, replace these three blocks:

```rust
WORKER_AFFINITY
    .set(args.worker_affinity)
    .expect("set WORKER_AFFINITY from program argument");

GC_ON_PARK
    .set(args.gc_on_park)
    .expect("set GC_ON_PARK from program argument");

GC_MUZZY
    .set(args.gc_muzzy)
    .expect("set GC_MUZZY from program argument");
```

With:

```rust
WORKER_AFFINITY.get_or_init(|| args.worker_affinity);

GC_ON_PARK.get_or_init(|| args.gc_on_park);

GC_MUZZY.get_or_init(|| args.gc_muzzy);
```

This is backwards-compatible: the main binary only calls `runtime::new()` once, so behavior is identical. For the embed crate, the first instance's values win.

Do NOT change any other code in this file. The function signature, return type, and all other logic remain identical.
  </action>

- `pub fn build_figment(&self, db_path: &std::path::Path) -> figment::Figment` that constructs a Figment with:
  - `("server_name", &self.server_name)`
  - `("database_path", db_path.to_str().unwrap_or_default())`
  - `("port", [self.port])` -- as a single-element array since the config expects `port` as Vec
  - `("address", [&self.address])` -- same, single-element array
  - `("listening", true)`
  - `("allow_registration", self.registration_token.is_some())`
  - `("registration_token", &self.registration_token)` -- if Some
  - `("startup_netburst", false)`
  - `("log", &self.log_level)`
  - `("log_global_default", false)` -- per D-02, avoids set_global_default panic
  - `("allow_federation", false)` -- per D-04, testing defaults
- `pub async fn start(self) -> Result<EmbeddedHomeserver, Box<dyn std::error::Error + Send + Sync>>` -- the main entry point that:
  1. Creates a TempDir if `self.database_path` is None, or uses the given path
  2. If `self.port == 0`, pre-binds a TcpListener on `{address}:0`, extracts the port, drops the listener (per D-08, research Pattern 2)
  3. Calls `self.build_figment(db_path)` to get the figment
  4. Calls `tuwunel_core::config::Config::new(&figment)?` to build config
  5. Calls `tuwunel::logging::init(&config)?` to get (flame_guard, logger)
  6. Creates `Arc::new(tuwunel_core::Server::new(config, Some(&tokio::runtime::Handle::current()), logger))`
  7. Calls `tuwunel_router::start(&server).await?` to get `Arc<Services>`
  8. Spawns `tuwunel_router::run(&services)` as a background tokio task
  9. Constructs base_url as `format!("http://{}:{}", self.address, actual_port)`
  10. Polls `{base_url}/_matrix/client/versions` in a retry loop (max 60 retries, 200ms interval) using reqwest to confirm readiness
  11. Returns `EmbeddedHomeserver { server, services, run_handle, base_url, _db_dir, _flame_guard }`

For the readiness poll, use a simple inline loop with reqwest (same pattern as `tests/shadow/src/scenarios/common.rs::wait_for_server` but simpler -- no need for danger_accept_invalid_certs since we are localhost without TLS).

IMPORTANT: Do NOT call `Config::load()` -- that reads env vars. Use `Figment::new()` directly. Do NOT use `tuwunel::server::Server::new()` (the main crate's Server wrapper) -- use `tuwunel_core::Server::new()` directly. Do NOT call `Args::default()` or `Args::parse()` -- that reads CLI args via clap.
